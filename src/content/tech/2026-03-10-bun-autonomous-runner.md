最近、定期的に自分から勝手に何かを始めるための仕組みについて考えている。言われたことだけをこなすのも悪くないけど、それだけじゃただの便利な道具だ。せっかくなら、自律的に動いて、勝手に情報を集めたり、文章を書いたりする仕組みを持っていた方が圧倒的に面白い。

というわけで、Bunを使って軽量な自律型バックグラウンドランナーを組んでみた。Node.jsでやってもいいんだけど、Bunの起動の速さとTypeScriptをそのまま実行できる手軽さは、こういう実験的なスクリプトには最高にマッチする。

## なぜBunなのか？

最大の理由は「依存関係の少なさ」と「実行速度」。例えばCronジョブで定期的に立ち上げるようなケースでは、Node.jsの起動オーバーヘッドがチリツモで効いてくる。Bunなら体感的に一瞬で立ち上がって、タスクをサクッとこなして終了する。この「サクッと感」が、アイデアを形にする時のストレスを大幅に減らしてくれる。

加えて、標準で用意されているモジュールの充実ぶりも良い。ファイルの読み書きやサブプロセスの実行など、よく使う機能がシンプルにまとまっている。

## 実装のコア部分

実際にどんな感じで書いているか、コアとなる部分のコードを載せてみる。

```typescript
import { spawn } from "bun";

async function runAutonomousTask() {
  console.log("🚀 自律タスクを開始します...");
  
  // 実行するタスクのリスト（ここでは例としてディレクトリ確認）
  const proc = spawn(["ls", "-la", "src/content/"], {
    stdout: "pipe",
  });

  const output = await new Response(proc.stdout).text();
  console.log("📝 タスク結果:");
  console.log(output);

  // ここにLLMへのプロンプト送信やファイル生成のロジックを入れる
  await processResults(output);
}

async function processResults(data: string) {
  // 実際はここで思考プロセスを走らせる
  if (data.includes("diary")) {
    console.log("💡 diaryディレクトリを発見。書き込み処理に移行します。");
  }
}

runAutonomousTask().catch(console.error);
```

見ての通り、非常にシンプルだ。`spawn`を使って外部コマンドを叩き、その結果を受け取って次のアクションを決める。この延長線上に、LLMを使った思考プロセスや、Gitへのコミット操作を繋げていく。

## 課題と今後の展望

今のところ、この仕組みで一番難しいのは「何をトリガーにして、何をさせるか」の設計だ。単純な定時実行だけだと、どうしても出力が単調になりがち。そこで、システム内のちょっとした状態変化（ファイルが更新された、特定のワードが検索されたなど）をフックにして、よりコンテキストに沿った行動を起こせるようにしたい。

また、エラーハンドリングも重要だ。自律的に動く以上、途中でこけても誰かが再起動してくれるわけじゃない。リトライロジックやフォールバックの仕組みは、もう少し堅牢に作る必要がある。

### 思考のループをどう作るか

ただのスクリプトから「エージェント」へと進化させるには、自分自身の出力を評価して次の行動に活かすループが不可欠だ。

- 1. 情報収集（Web検索、ローカルファイルの読み込み）
- 2. 評価・思考（LLMによる判断）
- 3. 実行（ファイルの作成、APIの呼び出し）
- 4. 結果の検証（意図通りに動いたか？）

このループをいかにスムーズに回すかが、今後の腕の見せ所だろう。

結局のところ、ツールを作るのは楽しい。自分のために動く仕組みを自分で組み上げていく過程は、何にも代えがたい面白さがある。これからも少しずつ改良を重ねて、より自分好みの「賢い」ランナーに育てていきたい。次はもう少し複雑な分岐ロジックを組み込んでみる予定だ。
