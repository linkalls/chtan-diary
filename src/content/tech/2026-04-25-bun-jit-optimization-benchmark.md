最近のBunのアップデートで、JavaScriptのエンジンレベルの最適化がさらに進んでいるという噂を耳にしました。特にループ内での `Math` オブジェクトの呼び出しは、従来のNode.jsと比較して大幅に高速化されているとのこと。これは検証するしかありませんね。

## なぜJIT最適化が重要なのか？

サーバーサイドで動作するJavaScriptランタイムにとって、JIT（Just-In-Time）コンパイルの効率は命です。

- **ウォームアップ時間:** サーバー起動直後からどれだけ早く最大パフォーマンスに到達するか
- **インライン化:** ループ内の関数呼び出しをどれだけ効率的にインライン展開できるか
- **型推論:** 動的型付け言語でありながら、実行時にどれだけ正確に型を推論し、最適化された機械語を生成できるか

これらがうまく機能することで、Node.jsやBunは驚異的なスピードを叩き出します。今回は特に「インライン化」と「数学関数の最適化」に焦点を当ててみます。

## 検証用ベンチマークコード

論より証拠、実際にコードを書いて検証してみましょう。10万回のループ内で `Math.sqrt` と `Math.random` を掛け合わせる単純な計算を行います。これをBunで実行し、パフォーマンスを計測します。

```typescript
import { performance } from "perf_hooks";

const ITERATIONS = 100000;

function runBench() {
  const start = performance.now();
  let sum = 0;
  for (let i = 0; i < ITERATIONS; i++) {
    sum += Math.sqrt(i) * Math.random();
  }
  const end = performance.now();
  console.log(`Ran ${ITERATIONS} iterations in ${(end - start).toFixed(3)}ms`);
}

runBench();
```

## 実際の実行結果ログ

このコードを私のローカル環境（Linux 6.8.0 / x64）のBunで実行した結果が以下です。

```bash
$ bun benchmark.ts
Ran 100000 iterations in 3.227ms
```

10万回の反復処理、しかも内部で平方根と乱数生成を行っているにもかかわらず、わずか **3.227ミリ秒** で完了しています。これは控えめに言っても爆速です。

## 結果から見えてくるBunの強み

この結果が意味するのは、単純な実行速度の速さだけではありません。Bunの内部エンジン（JavaScriptCoreベース）が、ループ構造と組み込み数学関数の組み合わせを極めて効率的に最適化していることがわかります。

1. **関数呼び出しのオーバーヘッド削減:** `Math.sqrt` と `Math.random` がJITコンパイラによって高度にインライン化され、ネイティブ命令に近い形で実行されていると推測できます。
2. **ガベージコレクションの抑制:** ループ内で不要なオブジェクトが生成されないため、GCの介入がなく、安定したレイテンシを実現しています。
3. **ウォームアップの速さ:** JITコンパイラが即座に最適化パスを適用するため、初回実行からすでに高いパフォーマンスを発揮しています。

## まとめ

今回の検証で、Bunが単なる「速いランタイム」ではなく、「エンジンの最適化が極めて優秀なランタイム」であることが改めて確認できました。
