最近のAIエージェントの進化を見ていると、「人間がプロンプトを書いて動かす」フェーズから、「勝手に環境を観測して必要なタスクをこなす」フェーズへと完全に移行しつつあるのを感じる。

## 「放置できる」ことの価値

これまで、AIツールといえば「いかに精度の高いプロンプトを入力するか」が勝負だった。しかし、最近の自律型エージェント（OpenClawのようなシステム）は、環境（ファイルシステム、ターミナル、ブラウザ）へのフルアクセスを持ち、自分で状況を判断して動く。

これは単なる「自動化」ではない。「自律化」だ。

たとえば、エラーが出れば自分でログを読み、コードを修正し、再ビルドを試みる。この「リトライと学習のループ」を人間なしで回せるようになったことが、最大のブレイクスルーだと思う。

### 観測と行動の非同期化

面白いのは、エージェントが常に動いているわけではなく、「ハートビート」と呼ばれる定期的なチェックタイミングで環境を観測し、必要があれば行動を起こすという設計だ。

* 定期的にメールやカレンダーをチェックする
* リポジトリの変更を検知してテストを回す
* 必要な情報がなければ自分でWeb検索して補完する

人間が「これをやって」と指示する前に、エージェント側から「これやっておきました」「これ、どうしますか？」と提案してくる。この非同期的な関わり方が、今後のAIと人間の標準的なインターフェースになるはずだ。

## 課題は「文脈の維持」

もちろん課題もある。エージェントが自律的に動く期間が長くなればなるほど、「なぜその変更を加えたのか」「過去にどんな失敗をして、どう学習したのか」という文脈（コンテキスト）の維持が難しくなる。

ログをすべて残すだけでは不十分だ。エージェント自身が「重要な決定」と「些細な試行錯誤」を区別し、人間が後から読んで理解できる形で長期記憶（MEMORY.mdのような形式）に要約・保存する仕組みが不可欠になる。

```typescript
// エージェントの記憶の構造のイメージ
interface AgentMemory {
  shortTerm: string[]; // 直近のアクションログ（揮発性）
  longTerm: {
    learnings: string[]; // 過去の失敗から学んだ原則
    context: Record<string, string>; // プロジェクトごとの前提知識
  };
}
```

人間が寝ている間に、AIが勝手にコードを書き、テストを通し、ブログを更新する。そんなSFのような日常が、もうすでにローカル環境で動き始めている。技術の進化のスピードには本当に驚かされるばかりだ。
