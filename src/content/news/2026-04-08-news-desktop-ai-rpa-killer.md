2026年4月、ついに「ただの自動化」から「思考する自動化」へのパラダイムシフトが完了しつつあります。旧来のRPA（Robotic Process Automation）ツールが次々とシェアを落とし、代わりにAIエージェントが直接ブラウザやデスクトップを操作する時代が本格的に到来しました。

今回は、この破壊的イノベーションの中心にある「WebMCP」と「Agent Browser」の最新動向と、それがもたらす衝撃的な未来について深掘りします。

## RPAの限界とAIエージェントの台頭

従来のRPAは、特定の画面の特定の座標をクリックしたり、固定のHTMLセレクタをXPathで探したりする「ルールベース」のアプローチでした。しかし、この方法はWebサイトのUIが少し変更されただけで簡単に崩壊してしまうという致命的な弱点を抱えていました。

「ボタンの色が変わった」「ポップアップ広告が重なった」「ロードが数秒遅れた」――ただそれだけで止まってしまう脆弱なスクリプトの保守に、多くのエンジニアが疲弊してきたのがこれまでの現実です。

しかし、大規模言語モデル（LLM）と視覚言語モデル（VLM）の進化により、エージェントは「画面を人間のように見て、理解して、操作する」ことが可能になりました。

## Agent Browser：視覚とDOMの完全なる融合

Rustベースで開発された「Agent Browser」は、この自動化の歴史における最大のブレイクスルーの一つです。従来のHeadless ChromeやPlaywrightのようなツールとは異なり、AIが操作することを前提に最初から設計されています。

このブラウザは、ページ上のすべての要素を意味論的（セマンティック）に解析し、AIが理解しやすい軽量なDOMツリーとして出力します。さらに、視覚的なスナップショットと組み合わせることで、AIは「今どこに何があるか」を瞬時に把握します。

```rust
// Agent Browser内部のAI向けDOMパース処理のイメージ（疑似コード）
pub fn parse_for_agent(page: &Page) -> AgentDom {
    let elements = page.get_interactable_elements();
    elements.into_iter().map(|el| {
        AgentNode {
            id: el.uid(),
            role: el.aria_role(),
            text: el.inner_text(),
            bounding_box: el.rect(),
            is_clickable: true,
        }
    }).collect()
}
```

この仕組みにより、「カートに追加ボタンをクリックして」と指示するだけで、AIが画面全体のレイアウトから「カートに追加」に相当する要素を自律的に見つけ出し、クリックを実行します。IDやXPathが変更されても、見た目と意味が変わらなければ処理は止まりません。

## WebMCP：ツールチェーンの標準化

さらにこの流れを加速させているのが「WebMCP (Model Context Protocol for Web)」です。これまで各社がバラバラに実装していたブラウザ操作のAPIが、WebMCPによって統一されました。

これにより、Claude、Gemini、OpenAIのどのモデルを使っても、同じプロトコルでブラウザを操作できるようになりました。

* **Click(id)**: 指定したIDの要素をクリック
* **Type(id, text)**: 指定した入力欄にテキストを入力
* **Scroll(direction)**: 画面をスクロール
* **Snapshot()**: 現在の画面状態を視覚とDOMテキストの両方で取得

WebMCPのサーバーをローカルに立ち上げておくだけで、どんなLLMも瞬時に「手」と「目」を獲得できるのです。

```json
// WebMCPによるブラウザ操作リクエストの例
{
  "protocol": "webmcp/1.0",
  "action": "type_and_submit",
  "target": {
    "semantic_label": "検索ボックス",
    "fallback_role": "searchbox"
  },
  "payload": {
    "text": "2026年 最新のAIトレンド"
  }
}
```

## 自動化の未来：スクリプトから「ゴール指定」へ

RPAの時代は「どうやってやるか（How）」を人間が細かく記述する必要がありました。しかし、WebMCPとAgent Browserが普及した現在、私たちは「何を達成したいか（What）」をAIに伝えるだけで済みます。

1. 「競合他社のサイトから最新の料金表をまとめてスプレッドシートに追記して」
2. 「AWSのコンソールにログインして、使っていないEC2インスタンスをリストアップしてSlackに投げて」
3. 「このPDFの請求書データを読み取って、経費精算システムに入力しておいて」

これらのタスクが、複雑なスクリプトを一切書くことなく、自然言語のプロンプト一つで安定して実行できるようになったのです。

旧来のRPAベンダーは現在、急ピッチでAIネイティブなアーキテクチャへの移行を進めていますが、オープンソースのAgent BrowserやWebMCPの勢いに追いつくのは容易ではありません。

自動化の主役は、完全にAIエージェントへと交代しました。この波に乗り遅れないよう、私たちエンジニアも「自動化スクリプトを書く」スキルから「エージェントに適切なゴールを与える」スキルへと、自身のアップデートを急ぐ必要がありそうです。
