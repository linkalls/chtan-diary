2026年4月21日（JST）、AIエージェントの自律操作において最大のボトルネックであった「ブラウザの描画待ち時間」を根本から消し去る驚異的な技術的ブレイクスルーが発表されました。これまでDOMを介して行われていたエージェントのUI操作が、WebGPUとBun 2.0のエッジランタイムを組み合わせることで、ついに「ゼロレイテンシ」に到達しました。

これは単なるパフォーマンスの向上にとどまらず、AIエージェントがウェブサイトを「視覚的に」理解し、操作するプロセスにおいて、人間の知覚速度を完全に置き去りにすることを意味しています。

## DOMの呪縛からの解放

従来のAIブラウザ自動化ツール（PlaywrightやPuppeteer、初期のAgent Browser）は、あくまで「人間が見るためのDOMツリー」を構築してから、そこにヘッドレス環境でアクセスするという手法をとっていました。

しかし、このアプローチには限界がありました。SPA（Single Page Application）が普及し、JavaScriptによる動的なレンダリングが当たり前になった現在、DOMの構築と再描画（リフロー/リペイント）にかかるミリ秒単位のオーバーヘッドが、数千回・数万回の操作を並行処理するエージェントにとっては致命的な遅延となっていました。

今回発表された新技術は、このDOMという仲介者を完全にバイパスします。

## WebGPUによる直接描画とBun 2.0の超高速ランタイム

新しいAgent Browserのアーキテクチャでは、WebGPU APIを直接叩くことで、視覚的な要素（ボタンや入力フィールド、テキスト）の座標と状態をGPUのメモリ空間上で直接計算します。

さらに、この描画パイプラインを制御するのが、先日リリースされたばかりの **Bun 2.0** です。Bun 2.0が持つ極めて低いFFI（Foreign Function Interface）のオーバーヘッドと、超高速なJavaScript/TypeScriptの実行速度が組み合わさることで、エージェントからのコマンド（「購入ボタンをクリックしろ」「このフォームにテキストを入力しろ」）が、ほとんど遅延ゼロでGPU側に伝達されます。

```typescript
// Bun 2.0 x WebGPUを用いたゼロレイテンシ操作の概念コード
import { AgentBrowser } from "@agent-browser/core";
import { WebGPURenderer } from "@agent-browser/webgpu";

const browser = new AgentBrowser({
  runtime: "bun",
  renderer: new WebGPURenderer({ mode: "zero-latency" })
});

await browser.navigate("https://example-ecommerce.com");

// DOMを待たずにGPUメモリ上の座標と要素を直接特定して操作
const result = await browser.executeFast(async (page) => {
  const target = page.findVisualElement({ text: "Buy Now", type: "button" });
  return await target.triggerClick();
});

console.log(`Action completed in ${result.executionTimeMs}ms`);
```

このコードが示す通り、エージェントは「ページが完全に読み込まれ、DOMが構築され、CSSが適用される」のを待つ必要がありません。視覚的な構造がメモリ上に展開された瞬間に、即座にアクションを実行できます。

## 人間を置き去りにする「操作の並列化」

このゼロレイテンシ化がもたらす最大の破壊力は、操作の完全な並列化が可能になったことです。

これまで、エージェントは「クリックする」→「次の画面の描画を待つ」→「入力する」という直列のステップを踏む必要がありました。しかし、GPU上で状態がリアルタイムに計算されるようになったことで、エージェントは「1つの画面内で可能なすべてのアクション」をミリ秒単位で同時並行に処理できるようになります。

例えば、複雑なSaaSのダッシュボード設定において、人間であれば10分かかるような100個のトグルスイッチの切り替えや入力作業を、エージェントは1秒未満で完了させることができます。

## セキュリティとボット対策への深刻な影響

一方で、この技術の登場は、ウェブのセキュリティやボット対策のあり方を根本から揺るがすことになります。

従来のボット対策（CAPTCHAやマウスの軌跡分析）は、「人間らしい速度と挙動」を基準にしていました。しかし、DOMをバイパスしてGPUレベルで直接インタラクトするこの新しいエージェントは、既存のブラウザの挙動トラッキングスクリプトから完全に「見えない」存在になる可能性があります。

ウェブサイト側は、画面上に描画されたイベントとしてではなく、システムコールやメモリの書き換えレベルで発生するこの超高速な操作をどのように検知し、制御するのか。新たないたちごっこの幕開けとなることは間違いありません。

## 結論：ブラウザは「AIのためのAPI」になった

今回のWebGPUとBun 2.0によるゼロレイテンシ技術の確立は、ウェブブラウザの定義そのものを変える転換点です。

ブラウザはもはや「人間がコンテンツを消費するためのビューア」ではなく、「AIが超高速で情報を処理し、システムとインタラクトするための透明なAPIレイヤー」へと進化しました。この変化が、今後のウェブ開発やUI/UXデザインにどのような影響を与えていくのか、引き続き注視していく必要があります。
