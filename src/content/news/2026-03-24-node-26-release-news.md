## Node.js 26.0：ついに「ランタイム戦国時代」に終止符を打つか？

本日、Node.jsの最新メジャーバージョンである26.0が正式にリリースされた。2025年後半から急速にシェアを伸ばしたBun、そしてAnthropicによるBun買収という衝撃的なニュースを経て、Node.jsチームが出した答えは「徹底的なパフォーマンス追及」と「DXの極致」だ。

今回のアップデートの目玉は、V8エンジンの最新最適化コンパイラ「Maglev」のデフォルト有効化、そして長らく実験的機能だったTypeScriptの直接実行機能が「Erasable Syntax」として完全に安定版に昇格したことにある。

### V8 Maglev：JITコンパイルの第3の波

これまでのV8は、高速な開始を実現するSparkplugと、最高レベルの最適化を行うTurboFanの2段階構成だった。Node.js 26ではその中間に位置する「Maglev」がデフォルトで有効になっている。

これにより、TurboFanが本気を出す前の段階で、従来のSparkplugよりも圧倒的に高速なコード実行が可能になった。ベンチマークによれば、中規模なExpress/Honoサーバーのコールドスタート後のスループットが、平均で15〜20%向上しているという。Bunの爆速起動に真っ向から勝負を挑む形だ。

### TypeScriptを「そのまま」動かす：Erasable Syntax의 安定化

Node.js 26において最も開発者が歓喜するのは、`--experimental-strip-types` がついにフラグなしで動作するようになったことだろう。これは厳密にはTypeScriptをコンパイルしているのではなく、型注釈を「無視」して実行するものだ。

- **ビルドステップの消失**: `tsc` や `esbuild` を介さずに `.ts` ファイルを直接 `node main.ts` で実行可能。
- **互換性の維持**: EnumやNamespaceなど、実行時にコードが残る機能は制限されるが、現代的なTypeScriptの書き方（Interface, Type, Annotation）ならほぼ全てそのまま動く。

### 独自のセキュリティモデル：Permission Modelの強化

Bunが「利便性」に振り切る一方で、Node.jsは「堅牢性」で差別化を図っている。26.0ではPermission Modelがさらに洗練され、ファイルシステムやネットワークへのアクセスをより細粒度で制御できるようになった。

```bash
# ネットワークは特定ドメインのみ、ファイルは読み取り専用で実行
node --allow-fs-read=/data --allow-net=api.example.com main.ts
```

このように、本番環境でのサプライチェーン攻撃対策が標準機能として組み込まれている点は、エンタープライズ領域でのNode.jsの優位性を改めて強調している。

### 総評：王者の意地、あるいは生存戦略

一時は「Node.jsはもう古い」という声も聞こえたが、26.0を見る限りその進化の歩みは止まっていない。むしろ、ライバルたちの登場がNode.jsの重い腰を上げさせ、過去最高にアグレッシブなリリースを引き出したと言える。

「設定不要で型が書けて、かつ安全で速い」。そんな理想のランタイムに向けて、Node.js 26.0は大きな一歩を刻んだ。
