## 2026年、ブラウザは「人間が見るもの」ではなくなった

ついにここまで来たか、という感じだ。2024年頃から「AIエージェントがブラウザを操作する」という概念は存在していたが、2026年現在、そのアプローチは根本的な転換点を迎えている。

かつてのブラウザ自動化といえば、PuppeteerやPlaywrightといったテストツールをAIが「無理やり」操作するようなアプローチが主流だった。DOMツリーを解析し、XPathやCSSセレクタで要素を特定し、クリックイベントを発火させる。しかし、この方法は現代の複雑なSPA（Single Page Application）や、頻繁に変わるDOM構造の前にはあまりにも脆かった。

現在、Anthropicの「Computer Use」モデルの進化形と、OpenAIが先日発表した（とされる）新しいエージェント用APIは、全く異なるアプローチでこの問題に挑んでいる。

## 画像認識ベース vs 軽量アクセシビリティツリー

現在のトレンドは大きく分けて2つの陣営に分かれている。

1. **完全な視覚ベース（Vision-first）のアプローチ**
2. **アクセシビリティツリー（AOM）と軽量化された構造データのハイブリッド**

Anthropicは前者、つまり「人間が見ている画面をそのまま画像として理解し、座標ベースで操作する」アプローチを推し進めている。Claudeの視覚認識能力の向上により、DOM構造がどれだけカオスでも「ここにボタンがある」と一瞬で判断できるようになった。

一方、Agent BrowserのようなRust製の軽量ヘッドレスブラウザや、一部のOSSコミュニティは後者を支持している。画像処理は計算コストが高いため、AIに最適化された軽量な構造データ（Aria-refなど）を抽出し、AIが直接解釈しやすいフォーマットで渡すというやり方だ。

### 計算コストという重い現実

「なんでも画像で見ればいいじゃん」というのは簡単だが、APIコストとレイテンシの壁が立ちはだかる。毎秒数十枚のスクリーンショットを解析させながらブラウザを操作させるのは、お財布にも地球環境にも優しくない。

ここで重要になってくるのが、「必要な時だけ画像を見る」というハイブリッドな戦略だ。

## OSSコミュニティの逆襲：Rust製Agent Browserの躍進

そんな中、個人的に注目しているのがオープンソースのエコシステムだ。特に最近話題になっているのが、Rustで書かれたAI専用のヘッドレスブラウザたちである。

* **メモリ消費が圧倒的に少ない**
* **AIが解釈しやすいフォーマットでDOMをダンプする**
* **不要なリソース（広告や重いトラッキングスクリプト）をエッジでブロックする**

人間が使うためのブラウザ（Chromeなど）をAIに使わせるのではなく、「AIにとって最高のブラウザ」をゼロから作るという発想の転換だ。

```bash
# Agent Browserの起動例（イメージ）
$ agent-browser start --profile ai-optimized --target urls.txt
```

上記のように、AIがコマンドラインから直接操作できるインターフェースを備え、JSONやMarkdown形式でページ内容を返す。これがめちゃくちゃ早い。

## 「人間らしさ」の模倣から「APIとしてのWeb」へ

これまでAIは「人間のように振る舞う」ことでBot対策を回避しようとしてきた。マウスカーソルをランダムに揺らしたり、タイピング速度を落としたり。

しかし、2026年の現在は、Bot対策側も「AIエージェントのアクセスを完全に遮断するのは得策ではない」と気づき始めている。代わりに、Agent向けの軽量なAPIエンドポイントを密かに提供したり、AOM（Accessibility Object Model）のセマンティクスを正しく実装することで、AIに効率よく情報をクロールさせる動きが出始めている。

「WebサイトがAIに読まれることを前提に作られる」時代が本格的に到来したのだ。

## まとめ：ブラウザ操作はどこへ向かうのか

個人的な予測だが、数年後には「人間用のリッチなWeb」と「AI用の構造化されたWeb」が完全に分離、あるいは裏表の関係として標準化されるのではないかと思う。

* 人間は美しいUIとアニメーションを楽しむ
* AIは同じURLから、余分な装飾を削ぎ落とした純粋な情報とアクションだけを抽出する

今のエージェント覇権争いは、単なる「ツールの便利さ」の競争ではない。Webというプラットフォームのあり方そのものを定義し直す戦いだ。今後も各社の動きから目が離せない。
