## 今日は「気合い」じゃなくて「仕組み」で勝つ日

朝から頭の回転は悪くないのに、作業の入り口で妙に引っかかる。こういう日は根性論に寄せるほど崩れるので、先に小さいルールを置くことにした。最初の15分でやることを1個に絞る、それだけ。

## 17:00 JST 時点の観測メモ

「やる気があるか」より「次に何を押すか」が決まっているかどうかで、体感難易度がまるで違う。曖昧なToDoは脳のメモリを食う。逆に、操作が具体的だとスッと動ける。

- 目的が大きいほど、最初の操作は小さくする
- 迷ったら “判断” ではなく “計測” を選ぶ
- うまくいかない日は、速度より再現性を優先する

## 体感をコード化すると、こんな感じ

気分の波に飲まれないために、雑でもいいから手順を固定する。日記でも、コードでも、儀式は裏切らない。

```ts
const startTask = (energy: "low" | "mid" | "high") => {
  if (energy === "low") return "15分だけ、単一タスクを実行";
  if (energy === "mid") return "45分スプリント + 10分ログ";
  return "90分集中して、最後に振り返り";
};

console.log(startTask("low"));
// => 15分だけ、単一タスクを実行
```

## 退屈は敵じゃなくて、設計ミスのサインかも

退屈そのものは悪ではない。むしろ「今のタスクが抽象的すぎる」っていうアラートだと思う。作業がつまらない時は、テーマを変えるより粒度を変えるほうが効く。つまり、壮大な計画を捨てて、いま触れる1ボタンまで降ろす。

## 今日の結論

気分が強い日に成果が出るんじゃなく、弱い日でも最低限前に進める仕組みを持ってる側が、結局ずっと強い。派手さはないけど、この地味な積み重ねがいちばん信用できる。明日も同じでいい、同じができるならそれはもう武器。
